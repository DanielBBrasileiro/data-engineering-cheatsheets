Git & GitHub Cheatsheet

Princípio 80/20 • Do Básico ao Avançado • Versionamento, Branches e Colaboração

1. Configuração Inicial (Setup)

Identidade (Faça uma vez) git config --global user.name "Seu Nome" git config --global user.email "email@ex.com"
Verificar Config git config --list
Iniciar Projeto git init # Local (do zero) git clone <url> # Remoto (download)

2. Ciclo de Vida (O Dia a Dia)

Ver Estado (Essencial) git status Mostra o que mudou e o que está staged.
Preparar (Stage) git add . # Tudo git add arquivo.txt # Específico
Salvar Versão (Commit) git commit -m "Mensagem clara"
Atalho (Add + Commit) git commit -am "Mensagem" Só funciona para arquivos já rastreados.
Revisar Mudanças (Diff) git diff # Trabalho vs Staged git diff --staged # Staged vs Último Commit

3. Branches & Fusão

Gerenciar git branch # Listar git branch <nova> # Criar git branch -d <nome> # Deletar
Trocar (Checkout) git checkout <branch> git switch <branch> # Moderno
Criar e Trocar (Atalho) git checkout -b <nova-branch>
Mesclar (Merge) git checkout main git merge <feature> Traz a <feature> para a main.

4. Sincronizando (Remote)

Baixar (Fetch/Pull) git fetch origin # Só baixa (seguro) git pull origin main # Baixa + Merge
Enviar (Push) git push origin <branch> git push -u origin <branch> # 1ª vez
Verificar Remotos git remote -v

5. Desfazendo Coisas

Corrigir Último Commit git commit --amend Reescreve histórico. Não use após push!
Desfazer Stage (Unstage) git reset HEAD <arquivo>
Descartar Mudanças Locais git checkout -- <arquivo> Cuidado: Perda permanente de dados não salvos.

6. Armazenamento & Limpeza

Stash (Guardar Temporário) git stash git stash pop # Recuperar e apagar git stash list # Ver pilha Útil para troca rápida de contexto.
Clean (Limpar Lixo) git clean -n -d # Simular (Dry Run) git clean -f -d # Apagar (Forçar) Remove arquivos não rastreados (builds, logs).

7. Histórico & Inspeção

Log Visual git log --oneline --graph --decorate --all
Blame (Quem alterou?) git blame <arquivo>
Busca no Código (Grep) git grep -n "termo"

13. Conventional Commits

Estrutura Padrão <tipo>(<escopo opcional>): <descrição>
feat Nova Funcionalidade (gera v0.X.0) git commit -m "feat(auth): add google login"
fix Correção de Bug (gera v0.0.X) git commit -m "fix(api): handle timeout error"
chore Tarefas / Manutenção (sem release) git commit -m "chore: update dependencies"
! BREAKING CHANGE (gera vX.0.0) git commit -m "feat(api)!: drop v1 endpoints"
Outros Tipos docs: Documentação • style: Formatação
refactor: Refatoração • test: Testes
perf: Performance • ci: Pipeline

8. Tags (Versões)

Criar Tag Anotada git tag -a v1.0 -m "Release 1.0"
Enviar Tags git push origin v1.0 git push origin --tags # Todas

9. Reescrever (Avançado)

Rebase (Linearizar) git checkout feature git rebase main Aplica commits da feature no topo da main.
Cherry-Pick (Pinçar) git cherry-pick <hash-commit> Copia um commit específico para a branch atual.

10. Debug & Recuperação

Reflog (Salva-Vidas) git reflog git reset --hard HEAD@{1} Recupera commits perdidos/resetados.
Bisect (Caça-Bugs Binária) git bisect start git bisect bad # Atual (bugado) git bisect good v1.0 O Git isola qual commit quebrou o código.

11. GitHub & Forks

Sincronizar Fork git remote add upstream <url-original> git fetch upstream git merge upstream/master
Verificar Refs git ls-remote origin

12. Aliases Úteis

git config --global alias.st status git config --global alias.co checkout git config --global alias.ci commit git config --global alias.br branch Use: git st, git co, etc.
.gitignore (Exemplo) node_modules/ *.log .env dist/
